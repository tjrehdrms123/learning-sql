**러닝 SQL.sql 주의 사항**

쿼리사이의 구분을 하기 위해 한 줄씩 띄웠습니다.
가령, Subquery의 경우 구문이 길고, Select 문이 다시 시작되어 쿼리가 이어진 것인지 구분이 어려울 수 있으므로,
띄워진 경우라면 하나의 SQL 문이며, 그렇지 않다면 별개의 Select 구문입니다.

각 Chapter별 구분은 ===== 표시를 하였으며, 터미널 서식이 적용된 모든 구문에 대해 추렸습니다.

SQL 문의 앞에 MySQL 프롬프트가 있는 것은 그대로 두었습니다. 원문을 그대로 적용하였으므로,
대부분 MySQL 프롬프트가 있지만, 일부는 프롬프트 없이 SQL 구문만 있는 것도 있습니다. 만약 프롬프트가 필요없다면 일괄 replace 하시면 될 듯 합니다.

MySQL 프롬프트를 그대로 둔 이유 중 하나는 마지막 장에서 Drill 에서 실행되는 것이 있습니다. 이때는 Drill 프롬프트가 명시되어 있습니다.
이 경우, MySQL에서 실행하면 에러가 나므로, 이에 대한 혼란을 방지하려는 것도 있습니다.

책을 보고 타이핑을 하는 것에 대한 편의를 위한 목적으로 알고 있습니다.
따라서, 책의 흐름상 설명을 위해 사용한 오류가 나는 구문이나, 마지막 학습 점검 실습에서의 SQL 문도 포함하였습니다.
